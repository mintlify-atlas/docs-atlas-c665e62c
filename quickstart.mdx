---
title: "Quick Start"
description: "Lint your first vector schema in under two minutes."
---

<Steps>
  <Step title="Install VectorLint">
    Install VectorLint globally using your preferred package manager:

    <CodeGroup>

    ```bash npm
    npm install -g vectorlint
    ```

    ```bash yarn
    yarn global add vectorlint
    ```

    ```bash pnpm
    pnpm add -g vectorlint
    ```

    </CodeGroup>

    Verify the installation:

    ```bash
    vectorlint --version
    ```
  </Step>

  <Step title="Create a sample schema">
    Create a `schemas/` directory and add a Pinecone-style schema file:

    ```json schemas/products.json
    {
      "index": "products",
      "dimension": 1536,
      "metric": "cosine",
      "metadata_fields": [
        { "name": "product_id", "type": "string", "required": true },
        { "name": "category", "type": "string", "required": true },
        { "name": "price", "type": "number", "required": false }
      ]
    }
    ```
  </Step>

  <Step title="Initialize config">
    Run `init` to generate a `.vectorlintrc.json` config file in the current directory:

    ```bash
    vectorlint init
    ```

    The generated config looks like this:

    ```json .vectorlintrc.json
    {
      "extends": "vectorlint:recommended",
      "include": ["schemas/**/*.json", "schemas/**/*.yaml"],
      "rules": {}
    }
    ```

    `vectorlint:recommended` enables a curated set of rules covering dimension validation, required field enforcement, and structural consistency.
  </Step>

  <Step title="Run the linter">
    Lint your schemas directory:

    ```bash
    vectorlint lint schemas/
    ```

    With the schema from step 2, you should see a clean pass:

    ```text
    ✔ schemas/products.json — 0 violations

    1 file linted, 0 violations found.
    ```
  </Step>

  <Step title="Introduce a violation">
    Change `"dimension": 1536` to `"dimension": 500` in `schemas/products.json`, then run the linter again:

    ```bash
    vectorlint lint schemas/
    ```

    VectorLint reports the violation with a rule name, line number, and a hint:

    ```text
    ✖ schemas/products.json
      Line 3: dimensions/model-mismatch — Dimension 500 is not a recognized output size for common embedding models. Expected one of: 384, 512, 768, 1024, 1536, 3072.
      (Hint: text-embedding-3-small outputs 1536 dimensions)

    1 file linted, 1 violation found.
    ```

    Revert the change to restore the passing state.
  </Step>
</Steps>

<Tip>
  Run `vectorlint lint --fix` to automatically apply safe fixes. For violations that require judgment (like choosing a different model), VectorLint shows a hint but won't auto-fix.
</Tip>
