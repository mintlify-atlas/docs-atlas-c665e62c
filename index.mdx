VectorLint is a command-line linting tool that validates your vector embedding schemas, index configurations, and AI/ML pipeline definitions. It integrates into your development workflow and CI/CD pipeline to catch common mistakes — like dimension mismatches, missing metadata fields, and schema inconsistencies — before they cause silent failures in production.

## How it works

VectorLint reads your vector schema definitions — JSON or YAML files — and runs a configurable set of rules against them. Each rule checks for a specific class of problem and reports violations with file paths, line numbers, and actionable fix suggestions.

<Steps>
  <Step title="Install VectorLint">
    Install globally via your preferred package manager or download a standalone binary.

    ```bash
    npm install -g vectorlint
    ```
  </Step>
  <Step title="Initialize your config">
    Run `vectorlint init` in your project root to generate a `.vectorlintrc.json` config file pre-configured for your schema files.

    ```bash
    vectorlint init
    ```
  </Step>
  <Step title="Lint your schemas">
    Run `vectorlint lint` to analyze your vector schemas and report any violations.

    ```bash
    vectorlint lint ./schemas
    ```
  </Step>
  <Step title="Integrate into CI">
    Add VectorLint to your CI pipeline. It exits with code `1` when violations are found, making it easy to block merges on schema errors.

    ```bash
    vectorlint lint --format json > vectorlint-report.json
    ```
  </Step>
</Steps>
